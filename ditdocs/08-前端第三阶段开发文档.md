# 前端第三阶段开发文档（体验治理与边界收敛期）

## 1. 阶段定位

第三阶段目标是在主流程可用基础上，统一视觉与动效语言，明确前后端资源边界。

## 2. 阶段目标

1. 统一图标与静态资源规范，减少跨端资源耦合。
2. 稳定关键页面动效与交互节奏。
3. 让模型生成流程在“有上游/无上游”两种场景都可运行。

## 3. 关键工作

### 3.1 视觉与组件治理

- 图标方案统一到 Iconify（`@iconify/vue`）。
- 样式以全局 Token 变量驱动，减少页面级颜色分叉。
- 复用卡片组件与页面模板，降低重复样式实现。

### 3.2 动效治理

- 页面入场、弹层、卡片流统一使用 GSAP 组织时序。
- 保留轻量 hover/focus 在 CSS 层，控制脚本动效复杂度。

### 3.3 能力边界

- 前端负责静态视觉资源和交互编排。
- 后端只返回业务数据与动态资源地址（如生成图 URL）。

### 3.4 模型流程兜底

- 通过 `/api/model/generate` 发起建模。
- 在上游未配置时，仍能消费后端 fallback 返回继续前端流程。

## 4. 阶段交付

| 交付项 | 说明 |
|---|---|
| 统一视觉语言 | 图标、配色、基础组件风格一致 |
| 动效规范落地 | 页面与弹层动效节奏可复用 |
| 资源职责清晰化 | 前后端对静态/动态资源边界明确 |
| 模型调用可降级 | 上游不可用时业务流程仍可继续 |

## 5. 验收口径

1. 主流程页面在不同主题下视觉一致、状态清晰。
2. 动效不会阻断交互，页面切换无明显残留。
3. 模型链路在降级模式可完成任务状态更新与结果展示。
4. 社区、工作区、我的内容在资源展示上职责不混用。

## 6. 后续输入

- 将页面级动效 preset 抽离为共享能力。
- 继续细化错误提示分级（网络异常/业务异常/参数异常）。
