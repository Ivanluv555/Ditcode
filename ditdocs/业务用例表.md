---

## 1. 角色定义

- `游客`：未登录用户，仅可浏览公开社区内容。
- `已登录用户`：可创建存档、输入建模指令、管理私密/发布状态。
- `系统`：前端业务编排 + mock 服务持久化层（非人工角色）。

## 3. 关键用例详细说明

## UC-003 首次输入触发新存档创建

- 目标：避免空存档污染，仅在用户真正开始工作时建档。
- 前置条件：用户已进入“新工作”状态，当前无激活存档或为空上下文。
- 触发：用户在底部输入区点击发送。
- 主流程：
  1. 前端校验输入（文本或首轮图片）。
  2. `useTaskStore.addTask()` 检测当前无存档时自动 `createArchive()`。
  3. 写入 `tasks/messages/modelAsset`（按当前阶段 mock 逻辑）。
  4. 调 `/api/workspace` 持久化。
- 后置条件：新存档落盘，成为 `activeArchiveId`。

## UC-005 非首轮隐藏上传入口

- 目标：每个存档仅首轮允许图文输入。
- 前置条件：当前存档已有至少一条消息。
- 主流程：
  1. 前端计算 `allowImageAttachment = false`。
  2. 隐藏上传按钮、文件名、预览与移除按钮。
  3. 若用户存在遗留已选图片，自动清理。
  4. 提交时强制忽略图片字段。
- 后置条件：后续轮仅文本输入。

## UC-008 打开社区项目并 fork 到个人存档

- 目标：社区内容不可直接覆盖，采用分支化编辑。
- 前置条件：社区存在公开项目。
- 主流程：
  1. 用户点击社区卡片。
  2. 前端携带 `source=community` 打开项目。
  3. store 执行 `openArchiveFromCommunity()` 创建新个人存档。
  4. 复制源项目上下文到新存档（messages/tasks/modelAsset）。
- 后置条件：用户进入自己的分支存档。

## UC-010 删除个人存档（弹窗确认）

- 目标：安全删除，避免误操作。
- 前置条件：用户拥有该存档。
- 主流程：
  1. 点击左侧历史项垃圾桶按钮。
  2. 弹窗确认“取消/确认删除”。
  3. 确认后调用 `deleteArchive()`。
  4. 同步写回 `/api/workspace`。
- 约束：仅 `ownerId == currentUser.id` 可删除。
- 后置条件：存档从个人列表移除，社区原件不变。

## UC-013 发布存档到社区

- 目标：将私密存档转为公开可见。
- 前置条件：当前用户是存档 owner。
- 主流程：
  1. 在“我的内容”点击“发布到社区”。
  2. 前端调用 `/api/community/publish`。
  3. 服务端写入/更新 `community.json`。
  4. 社区列表刷新显示该项目。
- 后置条件：`isPrivate=false`，并可被他人 fork/remix。

## 4. 业务规则（统一约束）

1. **存档创建规则**：仅首次提交输入并开始处理时创建存档。  
2. **单作品规则**：一个存档仅一个 `modelAsset`（后续在同存档持续修正）。  
3. **上传规则**：每存档仅首轮可上传图片，后续仅文本。  
4. **权限规则**：删除/发布/取消发布仅 owner 可执行。  
5. **可见性规则**：默认私密，发布后才进入社区。  
6. **分支规则**：社区打开即 fork，社区原件不可被个人删除覆盖。  
7. **计数规则**：基于社区来源完成 remix 时，社区 `remixCount +1`。

## 5. 异常与边界场景

| 场景 | 预期处理 |
|---|---|
| 未登录直接提交输入 | 弹登录提示，阻止入库 |
| 存档不存在仍被打开 | 允许按策略创建占位分支（当前 mock 支持） |
| 用户尝试删除他人存档 | 前端隐藏删除入口，服务端二次校验拒绝 |
| 发布接口失败 | 前端提示失败，保留原私密状态 |
| remix 计数写入失败 | 不影响本地编辑流程，记录 warning |

## 6. 验收建议（按阶段）

### P0 验收
- 新建不落空档、首输建档。
- 首轮可传图、后续禁图。
- 历史删除有确认弹窗且仅删除个人存档。

### P1 验收
- 发布/取消发布可切换社区可见性。
- 社区打开后能 fork 到个人存档。
- remix 后社区计数增长。

### P2 验收
- 已发布存档对话显示作者名。
- 社区与我的内容列表状态一致。
