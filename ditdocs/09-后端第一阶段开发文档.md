# 后端第一阶段开发文档（基础架构搭建期）

## 1. 阶段定位

第一阶段目标是完成后端工程骨架与公共能力，为业务模块迭代提供稳定底座。

## 2. 阶段目标

1. 建立 Spring Boot 工程与可运行环境。
2. 完成全局配置、异常处理、ID 生成、健康检查等核心能力。
3. 明确 `core` 与 `module` 分层边界。

## 3. 技术基线

- 框架：Spring Boot（WebMVC + Validation + Data JPA + Security）。
- 数据：MySQL/H2（运行时可切换）。
- 文档：SpringDoc OpenAPI。

## 4. 核心实现

1. `core/config`：CORS、安全策略、序列化配置、模型网关配置绑定。
2. `core/error`：统一异常结构（`{ ok: false, message }`）。
3. `core/auth`：Bearer Token 提取与当前用户解析。
4. `core/controller`：健康检查接口。
5. `core/util`：统一 ID 生成器。

## 5. 阶段交付

| 交付项 | 说明 |
|---|---|
| 工程底座 | Maven 工程、依赖、启动链路 |
| 分层约束 | `core` 管全局能力，`module` 管业务域 |
| 错误模型 | 全局异常拦截与统一响应格式 |
| 运维可观测入口 | `/api/health` 与 OpenAPI 基础配置 |

## 6. 验收口径

1. 服务可独立启动并通过健康检查。
2. 统一异常格式在参数错误和业务错误场景生效。
3. Token 解析能力可复用于各业务模块。

## 7. 阶段风险与输入

- 风险：仅有底座时业务规则尚未收敛，需在第二阶段补齐服务层约束。
- 输入：认证、工作区、社区三个业务模块设计稿与接口契约。
