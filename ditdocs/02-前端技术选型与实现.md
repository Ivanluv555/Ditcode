# 前端技术选型与实现

## 1. 技术栈选择

| 维度 | 方案 | 选择理由 |
|---|---|---|
| 框架 | Vue 3 (Composition API) | 组件组织清晰，状态与副作用可控 |
| 路由 | Vue Router 4 | 支持多页面业务域拆分 |
| 状态管理 | Pinia | 轻量、类型友好、适合业务 store |
| 构建工具 | Vite 5 | 启动快、构建快、开发体验好 |
| 动画 | GSAP + Vue3Lottie | 处理关键转场和欢迎动效 |
| 图标 | Iconify (`@iconify/vue`) | 统一来源、统一风格、减少本地图标碎片 |

---

## 2. 代码组织

```text
src/
  app/                 # 路由与应用入口
  packages/
    auth/              # 登录注册
    workspace/         # 创作主流程
    discovery/         # 社区
    content/           # 我的内容
    settings/          # 设置触发
  shared/
    components/        # 复用组件
    hooks/             # 可复用逻辑
    utils/             # 工具方法
    core/              # webgl/socket 等基础能力
```

---

## 3. 核心业务状态模型（Pinia）

### `useAuthStore`

- 负责 token 管理和登录态恢复
- 接口：`/api/auth/default-login|session|register|login|logout`

### `useTaskStore`

- 负责工作区聚合：archives/messages/tasks/modelAsset
- 负责社区相关接口：`/api/community/*`
- 负责模型生成：`/api/model/generate`

---

## 4. 关键交互流程

1. 用户输入 prompt（首轮可附图）
2. 前端调用 `/api/model/generate`
3. 返回后更新任务状态与模型资产
4. 同步 `/api/workspace` 持久化
5. 若来源于社区存档，调用 remix 计数接口

---

## 5. 资源边界约定

1. 图标统一 Iconify（禁止继续引入散落 SVG 图标文件作为 UI icon）。
2. 页面插图/封面等静态图仅由前端维护。
3. 后端返回动态资源地址（如模型生成图片 URL），前端只消费。

---

## 6. 后续建议

1. 对 `App.vue` 继续拆分（layout/topbar/sidebar/composer）减少单文件复杂度。
2. 为 `useTaskStore` 引入更细粒度 action（model/workspace/community 子域）。
3. 增加端到端测试覆盖关键路径（登录、创作、发布、remix）。
