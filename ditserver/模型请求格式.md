# DiT360 Server-to-Server API 规范

本规范用于服务器到服务器调用，不包含浏览器页面交互约定。

## 1. 概览

- Base URL: `http://<host>:<port>`
- 传输协议: HTTP
- 编码: UTF-8
- 请求体格式: `application/json`
- 图片输入: Base64 字符串
- 图片输出: `image/png` 二进制文件流

## 2. 接口定义

### 2.1 健康检查

- 方法: `GET`
- 路径: `/api/health`
- 成功状态码: `200`
- 成功响应体:

```json
{
  "ok": true,
  "service": "DiT360 API",
  "status": {
    "device": "cuda:0",
    "model_dir": "...",
    "lora_dir": "...",
    "output_dir": "...",
    "text_pipeline_loaded": false,
    "edit_pipeline_loaded": false
  }
}
```

### 2.2 全景图生成

- 方法: `POST`
- 路径: `/api/generate`
- 请求头: `Content-Type: application/json`
- 成功状态码: `200`
- 成功响应头:
  - `Content-Type: image/png`
  - `X-Task-Id: <task-id>`
  - `X-Result-Mode: image`
- 成功响应体:
  - PNG 文件二进制流

## 3. 请求模型

### 3.1 JSON Schema（描述性）

```json
{
  "type": "object",
  "required": ["prompt", "imageBase64"],
  "properties": {
    "prompt": {
      "type": "string",
      "minLength": 1,
      "description": "文本提示词"
    },
    "imageBase64": {
      "type": "string",
      "minLength": 1,
      "description": "输入图像 Base64，支持 data URL 前缀"
    },
    "seed": {
      "type": "integer",
      "description": "随机种子，可选"
    },
    "steps": {
      "type": "integer",
      "description": "推理步数，可选"
    },
    "guidanceScale": {
      "type": "number",
      "description": "引导系数，可选"
    }
  },
  "additionalProperties": true
}
```

### 3.2 字段约束

- `prompt`: 去除首尾空格后不能为空。
- `imageBase64`: 必填；支持以下两种格式:
  - 纯 Base64: `iVBORw0KGgo...`
  - Data URL: `data:image/png;base64,iVBORw0KGgo...`
- `seed`: 若提供，必须可解析为整数。
- `steps`: 若提供，必须可解析为整数。
- `guidanceScale`: 若提供，必须可解析为数字。

## 4. 错误响应模型

统一错误响应:

```json
{
  "ok": false,
  "error": {
    "code": "INVALID_FIELD_TYPE",
    "message": "steps 必须是整数",
    "details": {
      "field": "steps"
    }
  }
}
```

字段说明:

- `error.code`: 机器可读错误码。
- `error.message`: 人类可读错误描述。
- `error.details`: 可选扩展字段。

## 5. 错误码与状态码

- `INVALID_CONTENT_TYPE` -> `415`
  - 含义: 请求头不是 `application/json`。
- `INVALID_JSON` -> `400`
  - 含义: 请求体不是合法 JSON。
- `INVALID_JSON_SCHEMA` -> `400`
  - 含义: JSON 顶层不是对象。
- `MISSING_PROMPT` -> `400`
  - 含义: 缺少 prompt 或为空。
- `MISSING_IMAGE` -> `400`
  - 含义: 缺少 `imageBase64` 或为空。
- `INVALID_FIELD_TYPE` -> `400`
  - 含义: 可选参数类型不匹配。
- `INVALID_IMAGE_BASE64` -> `400`
  - 含义: `imageBase64` 无法 Base64 解码。
- `INVALID_IMAGE_CONTENT` -> `400`
  - 含义: 解码后不是合法图片文件。
- `PAYLOAD_TOO_LARGE` -> `413`
  - 含义: 请求体超过服务端最大限制。
- `INFERENCE_FAILED` -> `500`
  - 含义: 模型推理阶段失败。
- `INTERNAL_ERROR` -> `500`
  - 含义: 其他未预期错误。

## 6. 调用示例

### 6.1 curl

```bash
curl -X POST "http://127.0.0.1:5000/api/generate" \
  -H "Content-Type: application/json" \
  -d '{
    "prompt": "a sunset beach panorama",
    "imageBase64": "data:image/png;base64,iVBORw0KGgoAAA...",
    "seed": 123,
    "steps": 28,
    "guidanceScale": 2.8
  }' \
  --output panorama.png
```

### 6.2 Python requests

```python
import requests

payload = {
    "prompt": "a sunset beach panorama",
    "imageBase64": "data:image/png;base64,iVBORw0KGgoAAA...",
    "seed": 123,
    "steps": 28,
    "guidanceScale": 2.8,
}

resp = requests.post("http://127.0.0.1:5000/api/generate", json=payload, timeout=600)
resp.raise_for_status()
with open("panorama.png", "wb") as f:
    f.write(resp.content)
```

## 7. 兼容性说明

- 该接口当前定位为服务器间调用协议，不再保证 `multipart/form-data` 兼容。
- 建议调用方根据 `error.code` 做机器分流处理，不要只依赖 `message` 文本。
